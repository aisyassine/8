  1. Prérequis
    • Une machine virtuelle VMware avec PfSense déjà installé.
  2. Étapes
    1. Configuration des interfaces réseau dans VMware
      1. Ouvrez VMware et sélectionnez votre machine virtuelle PfSense.
      2. Cliquez avec le bouton droit de la souris sur la machine virtuelle PfSense et sélectionnez Settings.
      3. Dans l'onglet Hardware, cliquez sur Add pour ajouter de nouvelles interfaces réseau.
      4. Sélectionnez Network Adapter puis cliquez sur Next.
      5. Répétez ces étapes pour ajouter trois interfaces réseau : WAN, LAN et DMZ.
      6. Pour chaque interface, sélectionnez le mode de réseau approprié :
        • Pour l'interface WAN, sélectionnez Bridged (ce qui signifie que la VM partagera le réseau avec votre machine hôte).
        • Pour l'interface LAN et DMZ, sélectionnez Custom: Specific virtual network et assignez-le à un réseau virtuel différent pour chacun.
    2. Configuration des interfaces réseau dans PfSense
      1. Démarrez votre machine virtuelle PfSense.
      2. Une fois que PfSense est démarré, sélectionnez l'option Assign Interfaces dans le menu.
      3. Suivez les instructions à l'écran pour assigner les interfaces WAN, LAN et DMZ aux adaptateurs réseau que vous avez ajoutés précédemment.
      4. Après avoir assigné les interfaces, sélectionnez l'option Set interface(s) IP address.
      5. Configurez une adresse IP pour chaque interface :
        • Pour l'interface WAN, vous pouvez généralement obtenir une adresse IP automatiquement via DHCP. Sinon, vous devrez spécifier une adresse IP statique.
        • Pour les interfaces LAN et DMZ, spécifiez des adresses IP statiques dans des sous-réseaux différents.
      6. Une fois que vous avez configuré les adresses IP, votre PfSense devrait être prêt à être utilisé. Vous pouvez vérifier l'état des interfaces en sélectionnant l'option Interfaces dans le menu.
    3. C'est tout pour ce tutoriel. Dans les tutoriels suivants, nous aborderons la configuration du serveur DHCP, du serveur DNS, du pare-feu, du VPN, etc.
  1. Configuration de PfSense
    • Prérequis
      • Une machine virtuelle VMware avec PfSense déjà installé et configuré comme détaillé dans le Tuto1.
    • Étapes
      1. Activation et configuration du serveur DHCP
        1. Connectez-vous à l'interface web de PfSense à l'aide de l'adresse IP de l'interface LAN que vous avez configurée précédemment.
        2. Allez à Services > DHCP Server.
        3. Sélectionnez l'onglet correspondant à votre interface LAN.
        4. Cochez la case Enable DHCP server on LAN interface.
        5. Configurez la plage d'adresses IP que le serveur DHCP distribuera.
        6. Cliquez sur Save.
      2. Activation et configuration du serveur DNS
        1. Allez à Services > DNS Resolver.
        2. Cochez la case Enable DNS Resolver.
        3. Configurez les paramètres selon vos besoins.
        4. Cliquez sur Save.
      3. Création d'un deuxième compte administrateur "backup"
        1. Allez à System > User Manager.
        2. Cliquez sur +Add.
        3. Remplissez les champs du nouveau compte utilisateur : nom d'utilisateur "backup", mot de passe, et cochez la case Click to create a user certificate.
        4. Sous le champ Group Membership, ajoutez l'utilisateur au groupe admins.
        5. Cliquez sur Save.
      4. Activation et configuration de l'accès SSH
        1. Allez à System > Advanced.
        2. Sélectionnez l'onglet Admin Access.
        3. Sous Secure Shell, cochez Enable Secure Shell.
        4. Cliquez sur Save.
      5. Configuration de l'interface graphique en français
        1. Allez à System > General Setup.
        2. Sous Localization, sélectionnez Français pour Language.
        3. Cliquez sur Save.
      6. Configuration de l'accès à l'interface graphique depuis le WAN
        1. Allez à Firewall > Rules.
        2. Cliquez sur l'onglet WAN.
        3. Cliquez sur Add pour ajouter une nouvelle règle.
        4. Configurez la règle pour permettre l'accès à l'interface web de PfSense depuis le WAN.
        5. Cliquez sur Save.
  1. Installation et configuration des postes et serveurs
    • Prérequis
      • Une machine virtuelle VMware avec PfSense déjà installé et configuré comme détaillé dans le Tuto1 et Tuto2.
      • Les images ISO de Windows 10 et de la distribution Linux de votre choix.
    • Étapes
      1. Installation de Windows 10 sur les postes prévus
        1. Dans VMware, créez une nouvelle machine virtuelle.
        2. Sélectionnez l'image ISO de Windows 10 comme média d'installation.
        3. Suivez les instructions à l'écran pour installer Windows 10.
        4. Répétez ces étapes pour chaque poste Windows 10 requis.
      2. Installation de Linux sur le poste prévu
        1. Dans VMware, créez une nouvelle machine virtuelle.
        2. Sélectionnez l'image ISO de la distribution Linux de votre choix comme média d'installation.
        3. Suivez les instructions à l'écran pour installer Linux.
      3. Installation et configuration du serveur web dans la DMZ
        1. Créez une nouvelle machine virtuelle dans VMware pour le serveur web.
        2. Installez un système d'exploitation de votre choix (par exemple, Linux).
        3. Une fois le système d'exploitation installé, installez un serveur web (par exemple, Apache ou Nginx).
        4. Configurez le serveur web selon vos besoins.
      4. Installation et configuration du serveur de fichiers dans la DMZ
        1. Créez une nouvelle machine virtuelle dans VMware pour le serveur de fichiers.
        2. Installez un système d'exploitation de votre choix.
        3. Une fois le système d'exploitation installé, installez un serveur de fichiers (par exemple, Samba pour le partage Windows et NFS pour le partage Linux).
        4. Créez un partage NFS pour le poste Linux :
        5. Installez le serveur NFS (sudo apt install nfs-kernel-server pour Ubuntu/Debian).
        6. Créez un répertoire à partager (sudo mkdir /path/to/share).
        7. Éditez le fichier /etc/exports pour ajouter votre partage (/path/to/share *(rw,sync,no_subtree_check)).
        8. Lancez le serveur NFS (sudo systemctl start nfs-kernel-server).
        9. Créez un partage avec un quota de 1 Go :
        10. Installez le système de quotas (sudo apt install quota pour Ubuntu/Debian).
        11. Éditez le fichier /etc/fstab pour activer les quotas sur le système de fichiers de votre choix.
        12. Initialisez les quotas (sudo quotacheck -cug /path/to/filesystem).
        13. Activez les quotas (sudo quotaon /path/to/filesystem).
        14. Configurez le quota pour le répertoire partagé (sudo edquota -u user et définissez le quota à 1 Go).
  2. C'est tout pour ce tutoriel. Dans les tutoriels suivants, nous aborderons la configuration des accès et règles de pare-feu, du VPN, etc.
  1. Accéder à l'interface Web de PfSense
    1. Depuis un poste sur le réseau LAN, ouvrez un navigateur Web et entrez l'adresse IP de l'interface LAN de PfSense. Par défaut, l'adresse est http://192.168.1.1.
    2. Connectez-vous à l'aide du nom d'utilisateur admin et du mot de passe que vous avez défini lors de la configuration initiale de PfSense.
  2. Accéder à la page de gestion des règles de pare-feu
    1. Dans l'interface Web de PfSense, naviguez vers Firewall > Rules.
    2. Cliquez sur l'onglet LAN pour gérer les règles de pare-feu pour l'interface LAN.
  3. Créer une règle pour permettre l'accès au partage de fichiers
    1. Cliquez sur Add pour ajouter une nouvelle règle.
    2. Sous Action, sélectionnez Pass pour permettre le trafic qui correspond à cette règle.
    3. Sous Interface, assurez-vous que LAN est sélectionné.
    4. Sous Address Family, sélectionnez IPv4.
    5. Sous Protocol, sélectionnez TCP.
    6. Sous Source, assurez-vous que LAN net est sélectionné.
    7. Sous Destination, cliquez sur Single host or alias et entrez l'adresse IP de votre serveur de fichiers dans la DMZ.
    8. Sous Destination port range, sélectionnez le port utilisé par votre serveur de fichiers pour le partage de fichiers.
    9. Sous Description, entrez une description pour la règle.
    10. Cliquez sur Save pour enregistrer la règle.
    11. Cliquez sur Apply changes pour appliquer la nouvelle règle.
  1. Installation du paquet OpenVPN
    1. Connectez-vous à l'interface Web de PfSense (par défaut http://192.168.1.1) avec le nom d'utilisateur admin et le mot de passe que vous avez défini.
    2. Naviguez vers System > Package Manager.
    3. Allez dans l'onglet Available Packages.
    4. Recherchez "openvpn-client-export" et cliquez sur Install à côté. Cela installera OpenVPN et un outil utile pour exporter les configurations client.
  2. Configuration d'OpenVPN
    1. Naviguez vers VPN > OpenVPN.
    2. Allez dans l'onglet Wizards.
    3. Sélectionnez le type de serveur Local User Access puis cliquez sur Next.
    4. Pour Certificate Authority, cliquez sur +Add New CA et remplissez les informations requises, puis cliquez sur Save.
    5. Pour Server Certificate, cliquez sur +Add New Cert et remplissez les informations requises, puis cliquez sur Save.
    6. Dans la configuration du serveur VPN, sélectionnez WAN pour l'interface, UDP on IPv4 only pour le protocole, choisissez un port pour le service (par défaut 1194), et pour TLS Configuration, cochez Use a TLS Key.
    7. Continuez à configurer selon vos besoins, puis cliquez sur Next.
    8. Ajoutez un utilisateur local pour le VPN, puis cliquez sur Next et Finish.
  3. Configuration des règles de pare-feu pour le VPN
    1. Naviguez vers Firewall > Rules.
    2. Allez dans l'onglet WAN.
    3. Cliquez sur Add pour ajouter une nouvelle règle.
    4. Configurez la règle selon les spécifications, puis cliquez sur Save et Apply Changes.
  4. Exportation de la configuration client OpenVPN
    1. Naviguez vers VPN > OpenVPN.
    2. Allez dans l'onglet Client Export.
    3. Trouvez le nom d'utilisateur que vous avez créé pour le VPN et téléchargez le fichier de configuration.
  1. Génération d'une paire de clés avec PowerShell pour le compte "admin"
    1. Ouvrez PowerShell et entrez la commande suivante pour générer une paire de clés RSA :
      2. ssh-keygen -t rsa
    2. Lorsqu'on vous demande d'entrer un fichier dans lequel sauvegarder la clé, vous pouvez appuyer sur Enter pour accepter le chemin par défaut (C:\Users\YourUserName\.ssh\id_rsa).
    3. Vous serez ensuite invité à entrer une passphrase. Vous pouvez choisir d'entrer une passphrase pour plus de sécurité ou laisser le champ vide pour ne pas en avoir.
    4. Une fois la clé générée, vous pouvez afficher la clé publique avec la commande suivante :
      5. Get-Content C:\Users\YourUserName\.ssh\id_rsa.pub
  2. Génération d'une paire de clés avec PuTTY pour le compte "backup"
    1. Ouvrez PuTTYgen (généralement installé avec PuTTY).
    2. Sélectionnez RSA comme type de clé à générer.
    3. Cliquez sur Generate et suivez les instructions pour générer la clé.
    4. Une fois la clé générée, vous pouvez entrer une passphrase facultative dans le champ Key passphrase pour plus de sécurité.
    5. Cliquez sur Save public key et Save private key pour sauvegarder vos clés.
    6. Copiez le contenu de la zone "Public key for pasting into OpenSSH authorized_keys file" - c'est votre clé publique.
  3. Association des clés avec les comptes correspondants sur PfSense
    1. Connectez-vous à l'interface Web de PfSense.
    2. Naviguez vers System > User Manager.
    3. Cliquez sur le nom d'utilisateur admin, puis sur Edit.
    4. Faites défiler jusqu'à SSH Key et collez votre clé publique générée avec PowerShell dans la zone de texte, puis cliquez sur Save.
    5. Répétez la même opération pour le compte backup, en utilisant la clé publique générée avec PuTTY.
  1. Vérification de la configuration et des règles de pare-feu
    1. Connectez-vous à l'interface Web de PfSense.
    2. Naviguez vers Firewall > Rules.
    3. Vérifiez que les règles de pare-feu pour les interfaces WAN, LAN et DMZ sont correctement configurées selon vos besoins. Les règles doivent permettre l'accès aux services et partages de fichiers, ainsi que l'accès VPN.
  2. Tests d'accès et de fonctionnement des services et partages de fichiers
    1. Depuis le poste Windows 10, tentez d'accéder au partage de fichiers sur le serveur dans la DMZ. Si vous ne pouvez pas y accéder, vérifiez la configuration du serveur de fichiers et les règles de pare-feu.
    2. Depuis le poste Linux, tentez d'accéder au partage NFS sur le serveur dans la DMZ. Si vous ne pouvez pas y accéder, vérifiez la configuration du serveur de fichiers et les règles de pare-feu.
    3. Depuis un poste sur le réseau WAN, tentez d'accéder au serveur Web dans la DMZ. Si vous ne pouvez pas y accéder, vérifiez la configuration du serveur Web et les règles de pare-feu.
  3. Tests de connectivité et de performance du VPN
    1. Configurez un client OpenVPN sur un poste externe avec le fichier de configuration exporté précédemment.
    2. Tentez de vous connecter au VPN. Si vous ne pouvez pas vous connecter, vérifiez la configuration d'OpenVPN et les règles de pare-feu.
    3. Une fois connecté au VPN, tentez d'accéder au réseau LAN de PfSense. Vous devriez être en mesure d'accéder aux postes et aux serveurs sur le réseau LAN.
    4. Testez la performance du VPN en transférant des fichiers entre le poste VPN et les postes sur le réseau LAN. Vous pouvez utiliser un outil comme iperf pour cela.
